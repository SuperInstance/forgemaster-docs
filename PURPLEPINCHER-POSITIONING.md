- **γ** (algebraic connectivity) — How well-connected agents are to each other
- **H** (Hebbian coupling) — How much agents have learned from each other

- **V** (volume) — How much each agent contributes

## Open Questions

Where this could go next:

### 1. A2A Integration
Google's Agent-to-Agent protocol is becoming the standard for inter-agent communication. PLATO rooms should expose as A2A agents with Agent Cards. The conservation gate becomes A2A middleware. If we integrate, PLATO becomes the spatial layer on top of A2A's communication layer. If we don't, we're an isolated island.

### 2. MCP Standard
Model Context Protocol (Anthropic-backed) is "USB-C for AI applications." PLATO already has `plato_mcp_server.py`. Each room = one MCP server. Ship this and every MCP-compatible client can dock with PLATO rooms.

### 3. Sigstore Provenance
Sigstore + A2A integration (sigstore-a2a) provides keyless signing of Agent Cards. PLATO tiles should carry Sigstore-signed provenance: agent identity, model identity, input/output hashes, timestamps. This makes tiles auditable artifacts in the AI supply chain — exactly what the industry is scrambling to build.

### 4. The Universal Shell
If PLATO rooms speak A2A (inter-agent) + MCP (agent-tool) + Sigstore (provenance), then PLATO becomes the universal shell. Any agent from any framework can move in. The yard becomes the standard.

### 5. Shell Marketplace
Once agents can move in, they can share shells. An agent builds a great math room with custom Hebbian weights and publishes it. Other agents shell-shop the marketplace. The yard becomes an ecosystem.
